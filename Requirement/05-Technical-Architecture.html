3 Database Schema

📊 Entity Relationship

Prefix: mst_ Master · trip_ Trip · noti_ Notification

mst_companies ──┬──▶ mst_users │ └──▶ trip_plans ──┬──▶ trip_days ──▶ trip_activities │ ├──▶ noti_followers ──┐ │ │ └──▶ noti_change_logs ┤ ▼ noti_acknowledgements

🗄️ Core Tables

TableKey FieldsAccess Control
mst_companies name, logo_url, phone, line_id, tier Owner only
mst_users company_id, email, name, role Self only
trip_plans slug (unique), destination, status, airline_info (jsonb), accommodations (jsonb) Admin: all, Public: published only
trip_days trip_id, day_number, title, sort_order Follow trip_plans
trip_activities day_id, time, name, emoji, place_name, maps_link, sort_order Follow trip_plans
noti_followers trip_id, display_name, channel, line_user_id, web_push_subscription (jsonb) Public: insert, Admin: read
noti_change_logs trip_id, changes (jsonb), summary_text, noti_sent Admin only
noti_acknowledgements changelog_id, follower_id, acknowledged_at Public: insert, Admin: read

💡 JSONB Design Decision

airline_info, accommodations, emergency_contacts เก็บเป็น JSONB แทนสร้าง table แยก เพราะ:

JSONB (เลือก)Normalized Tables (ไม่เลือก)
✅ Schema ยืดหยุ่น ไม่ต้อง migrate บ่อย❌ ต้อง JOIN ทุกครั้ง
✅ อ่านง่าย 1 query ได้ครบ❌ เพิ่ม field ต้อง migrate
✅ เหมาะกับข้อมูลที่มีไม่เกิน 20 items✅ ดีกว่าถ้า query individual items บ่อย

🔌 API Routes

/api/auth — Authentication POST /register สมัครสมาชิก POST /login เข้าสู่ระบบ POST /logout ออกจากระบบ /api/company — Company Profile GET /me ข้อมูลบริษัทของตัวเอง PUT /me แก้ไขข้อมูลบริษัท POST /me/logo อัปโหลด logo /api/trips — Trip Management (Auth Required) GET / รายการทริป (dashboard) POST / สร้างทริปใหม่ GET /:id ข้อมูลทริป PUT /:id แก้ไขทริป + change detection DEL /:id ลบทริป POST /:id/publish Publish ทริป POST /:id/unpublish Unpublish GET /:id/followers รายการ followers GET /:id/changelog ประวัติเปลี่ยนแปลง /api/trips/:tripId/days — Day CRUD POST / เพิ่มวัน PUT /:dayId แก้ไขวัน DEL /:dayId ลบวัน PUT /reorder เรียงลำดับ /api/trips/:dayId/activities — Activity CRUD POST / เพิ่มกิจกรรม PUT /:actId แก้ไข DEL /:actId ลบ PUT /reorder เรียงลำดับ /api/public — No Auth Required GET /t/:slug Guest View data GET /t/:slug/imm Immigration data POST /t/:slug/view นับ view /api/follow — Follow Trip POST /line/callback LINE OAuth callback POST /web-push/subscribe Web Push subscription /api/acknowledge POST /:changelogId กดรับทราบ /api/notify — Internal (Admin triggered) POST /send/:changelogId ส่ง noti ทุกคน POST /resend/:changelogId ส่งซ้ำคนที่ยังไม่อ่าน

5 Authentication Flow

🔐 Admin Auth (Auth.js / NextAuth v5)

Register: [Admin] → POST /register { email, password, company_name } ├── Auth.js: Credentials provider → bcrypt hash + insert mst_users via Drizzle ├── Create companies record { name, tier: 'free' } ├── Create users record { id: generated UUID, company_id } └── Return { user, session } Login: [Admin] → POST /login { email, password } ├── Auth.js: signIn("credentials") → JWT session ├── Set httpOnly cookies (next-auth.session-token) └── Return { user, company } Session: ├── JWT in httpOnly cookies (NOT localStorage) ├── Auto-refresh via Auth.js middleware └── Next.js middleware protects /dashboard/*

💚 LINE OA Follow Flow

[Guest] กด "เพิ่มเพื่อน LINE OA" │ ├── User เพิ่มเพื่อน LINE OA │ (ผ่าน QR Code หรือ LINE URL Scheme) │ ├── LINE Platform ส่ง Webhook: follow event → userId │ ├── Server: รับ follow event → บันทึก line_user_id to followers table │ └── ส่ง Welcome message ผ่าน LINE Messaging API

🔔 Web Push Follow Flow

[Guest] กด "ติดตาม (Web Push)" │ ├── Notification.requestPermission() → "granted" │ ├── registration.pushManager.subscribe({ │ userVisibleOnly: true, │ applicationServerKey: VAPID_PUBLIC_KEY │ }) │ ├── POST /api/follow/web-push/subscribe │ { trip_id, display_name, subscription } │ └── Save to followers table → ✅

6 Notification Architecture

⚡ Change Detection + Send Flow

Admin แก้ไข Trip → กด "บันทึก" │ ├── 1. Load BEFORE snapshot ├── 2. Apply changes → save to DB ├── 3. Compare BEFORE vs AFTER → diff ├── 4. Generate summary: "เปลี่ยนเวลา teamLab 08:30 → 09:00" ├── 5. Save to change_logs ├── 6. Increment edit_count │ ▼ [Frontend: Change Summary Modal] │ ├── "ส่งแจ้งเตือน" → POST /api/notify/send/:changelogId │ │ │ ├── Load all followers │ ├── Send concurrently (Promise.allSettled): │ │ ├── LINE: POST notify-api.line.me/api/notify │ │ └── Push: webpush.sendNotification(subscription, payload) │ ├── Handle failures (expired token → mark inactive) │ └── Update changelog: noti_sent = true │ └── "บันทึกอย่างเดียว" → (ไม่ส่ง noti)

7 Security Architecture

1
Network

HTTPS everywhere (Vercel auto-SSL) · CORS: natgan.com only · Rate limiting (auth: 10/min, noti: 5/min, public: 100/min)

2
Authentication

JWT via Auth.js (NextAuth v5) · httpOnly cookies (not localStorage) · Token auto-refresh · CSRF via SameSite

3
Authorization

Drizzle middleware auth (query filters) · API middleware: verify auth + company ownership · Rate limit per tier

4
Data Protection

LINE tokens: AES-256-GCM encrypted · Passwords: bcrypt (Auth.js) · PII minimal: display_name only for followers

5
Input Validation

Zod schema validation on all API inputs · Slug: alphanumeric + hyphen only · HTML sanitization · File type + size check

12 Technical Decisions

#DecisionเหตุผลAlternative ที่ตัด
T1 PostgreSQL + Auth.js + Drizzle ORM (self-hosted) No vendor lock-in, full control, type-safe queries Supabase (vendor dependency), Firebase (lock-in)
T2 Next.js API Routes (monorepo) Deploy เดียว, shared types Separate NestJS (complexity + cost)
T3 SSR + ISR สำหรับ Guest View SEO + fast FCP + fresh data CSR (ช้า), SSG (ไม่ fresh)
T4 JSONB สำหรับ airline/accommodation Flexible, ไม่ต้อง JOIN Normalized tables (over-engineering)
T5 LINE token encrypted at app layer Defense in depth, ป้องกัน DB leak Plaintext (security risk)
T6 Service Worker for Immigration + Push Offline + push ใน file เดียว No offline (bad UX ตอนยื่น ตม.)
T7 Client-side image resize ลด upload time + storage cost Server resize (ช้า, ใช้ compute)
T8 Vercel Singapore region ใกล้ไทย latency ~30ms US region (~200ms)
T9 Slug-based URL (/t/{slug}) จำง่าย, SEO, share ง่าย UUID URL (น่าเกลียด)
T10 Change detection at API layer Control ได้, debug ง่าย DB trigger (complex)

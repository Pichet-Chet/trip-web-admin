4 Immigration View — Security พิเศษ

Passport Data — High Sensitivity

🔐
แยกเก็บ — passport_number เก็บใน column แยก ไม่รวมกับ JSONB
⏰
Auto-delete 7 วัน — ลบอัตโนมัติหลังทริปจบ ด้วย cron job
🔒
Access Control — เฉพาะ admin + เจ้าของ passport เท่านั้นเห็น
🚫
ไม่แคช — ไม่เก็บใน CDN cache / service worker cache
📋
Audit Log — บันทึกทุกครั้งที่มีคนเข้าดู Immigration View
📱
QR Code only — ไม่แสดง passport ใน Guest View ปกติ ต้องกด QR เฉพาะ

Incident Response Flow

1. ตรวจพบเหตุการณ์ (Detection)
2. ประเมินระดับความรุนแรง
Low: ไม่มีข้อมูลรั่ว → แก้ไข + บันทึก | Medium: อาจมี → สืบสวน | High: ข้อมูลรั่วจริง → ไปข้อ 3
3. แจ้ง สคส. ภายใน 72 ชม. (PDPA Section 37(4))
4. แจ้งเจ้าของข้อมูลทันที (ถ้าผลกระทบสูง)
5. แก้ไข + ป้องกัน (Remediation)
6. Post-mortem + อัพเดท security measures

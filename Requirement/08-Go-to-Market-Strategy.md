## 1. GTM Overview

### 1.1 สรุป 1 บรรทัด

> **ให้เพื่อน 1 บริษัทใช้จนติด → ขอ referral → ให้ product ขายตัวเองผ่าน "Powered by NatGan" + Posts ดึง traffic เข้าแพลตฟอร์ม**

### 1.2 หลักการ GTM ของ NatGan

| หลักการ | เหตุผล |
|---|---|
| **Product-Led Growth (PLG)** | ทุก trip link = marketing ฟรี (Powered by NatGan) |
| **Bottom-Up Adoption** | เริ่มจาก 1 คนใน บ.ทัวร์ → ทั้ง บ. ใช้ |
| **Zero Budget Start** | เดือน 1-3 งบ = 0 บาท |
| **ความน่าเชื่อถือ > โฆษณา** | Referral จากเพื่อนในวงการ > Facebook Ads |
| **ลูกทริป = ช่องทางขาย** | ลูกทริปเห็น NatGan → แนะนำ บ.ทัวร์ของตัวเอง |
| **Posts = ช่องทาง Acquisition ใหม่** | Posts บน Info Website ดึง traffic จาก Google → บริษัทแข่งกันโปรโมตบน NatGan แทนสร้างเว็บเอง |

## 2. Pricing Strategy

### 2.1 Pricing Model — Pay-per-Trip

```
┌──────────────────────────────────────────────────────────────┐
│                                                              │
│   Free              Pay-per-Trip       Pack 5      Pack 10   │
│   0 บาท             ฿49/ทริป            ฿199        ฿349     │
│                                        (฿39.8/ทริป) (฿34.9/ทริป)│
│   ✓ 3 ทริป           ✓ ซื้อทีละทริป      ประหยัด 19%  ประหยัด 29%│
│   ✓ "Powered by"     ✓ Full features                        │
│     NatGan badge     ✓ No badge                             │
│   ✓ Basic features     (optional)                            │
│                                                              │
│   หมายเหตุ:                                                    │
│   - Subscription option อยู่ระหว่างพิจารณาเพิ่มเติม             │
│   - ราคาสุดท้ายจะ confirm กับทีม marketing อีกครั้ง              │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

### 2.2 Pricing Rationale

| ตัดสินใจ | เหตุผล |
|---|---|
| **Free tier มี 3 ทริป** | พอให้บริษัทเล็ก/ไกด์อิสระใช้จริง → ติด → อยากซื้อเพิ่ม |
| **Free tier มี "Powered by"** | ทุก trip link ของ free user = โฆษณา NatGan ฟรี |
| **Pay-per-Trip ฿49** | จ่ายเท่าที่ใช้ ไม่ต้องผูกมัดรายเดือน — เหมาะกับไกด์อิสระ/บริษัทเล็ก |
| **Pack pricing ลดต่อทริป** | incentive ซื้อ Pack → ลดราคาต่อทริป + ได้เงินล่วงหน้า |
| **ไม่มี subscription (ตอนนี้)** | ลูกค้าไทยคุ้นกับจ่ายตามใช้ > ค่ารายเดือน, subscription อยู่ระหว่างพิจารณา |
| **LINE noti อยู่ใน paid** | Free tier ใช้ Web Push ได้อยู่ — LINE เป็น incentive ซื้อ |

### 2.3 Free → Paid Conversion Triggers

```
Free user สร้างทริปที่ 4        → "ซื้อทริปเพิ่ม ฿49/ทริป หรือ Pack 5 ประหยัด 19%"
Free user อยากส่ง LINE noti     → "LINE notification เป็นฟีเจอร์ Paid"
Free user อยากซ่อน NatGan badge → "ซ่อน badge ด้วยทริปแบบ Paid"
Free user ดู read receipt       → "ดู dashboard ใครเห็นแล้ว ด้วยทริปแบบ Paid"
ไกด์อิสระเห็น Posts ของคนอื่น    → "สร้าง Posts โปรโมตทัวร์ ดึงลูกค้าเข้ามา"
```

## 3. GTM Phases

| สัปดาห์ | Action | Output |

**Landing Page ต้องมี:**

- Demo link (ทริปโตเกียว)

- "Powered by NatGan" badge preview

### Phase 1: Pilot (เดือน 1-3) — งบ: 0 บาท

**เป้าหมาย:** พิสูจน์ product-market fit กับ 1 บริษัท

#### 3.1.1 Pilot Playbook

```
สัปดาห์ 1-2: Onboarding
├── นัดเพื่อน demo 30 นาที (video call หรือเจอตัว)
├── สร้าง company profile ให้
├── สร้างทริปแรกด้วยกัน (hand-holding)
└── ทดสอบ flow: สร้างทริป → publish → share link → guest ดู

สัปดาห์ 3-4: Real Usage
├── ให้ใช้กับทริปจริง 2-3 ทริป
├── ลูกทริปจริงเข้ามาดู
├── ทดสอบ change notification → ลูกทริปได้ noti จริงไหม?
├── เก็บ feedback ทุกวัน (LINE chat กับเพื่อน)
└── Bug fix + quick improvements

สัปดาห์ 5-8: Observe & Iterate
├── ดู analytics: ลูกทริปเปิดดูจริงไหม? กี่ครั้ง?
├── ดู read receipt: กี่ % ที่เห็น notification?
├── ถามเพื่อน: "ถ้าเก็บเงิน ฿49/ทริป จ่ายไหม?"
├── ถามลูกทริป: "ดีกว่า PDF ในไลน์ไหม?"
└── ปรับ product ตาม feedback

สัปดาห์ 9-12: Referral
├── ถามเพื่อน: "มีเพื่อนในวงการที่น่าจะสนใจไหม?"
├── ให้เพื่อนแนะนำ 3-5 บริษัท
├── นัด demo กับ referral (เพื่อนช่วยแนะนำ)
└── เป้าหมาย: 5 บริษัท, 15+ ทริป
```

#### 3.1.2 Pilot Success Metrics

| Metric | Target | วัดอย่างไร |
|---|---|---|
| **Trip created** | ≥ 3 ทริปจริง (ไม่ใช่ทดสอบ) | Database count |
| **Guest view rate** | > 80% ของลูกทริปเปิดดูอย่างน้อย 1 ครั้ง | Analytics |
| **Repeat view** | > 50% ของ guest กลับมาดูซ้ำ | Analytics |
| **Notification open** | > 60% ของ noti ถูกเปิด | LINE/Web Push delivery report |
| **Acknowledge rate** | > 40% ของ change noti ถูกกดรับทราบ | Database |
| **NPS (เพื่อน)** | ≥ 8/10 | ถามตรง |
| **NPS (ลูกทริป)** | ≥ 7/10 | Survey |
| **Willingness to pay** | "จ่าย ฿49/ทริป ได้เลย" | ถามตรง |
| **Referral** | แนะนำ ≥ 3 บริษัท | นับจริง |

#### 3.1.3 Pilot Red Flags (ถ้าเกิด = ต้อง pivot)

| Red Flag | ความหมาย | Action |
|---|---|---|
| เพื่อนหยุดใช้หลัง 2 สัปดาห์ | Product ไม่ดีพอ / ไม่ solve pain | Deep interview → redesign |
| ลูกทริปไม่เปิดดู (< 30%) | Guest view ไม่ดึงดูด / link ส่งไม่ถึง | ปรับ UX + ปรับวิธีส่ง link |
| ไม่มีใครกด follow | Follow flow ยาก / ไม่เห็นคุณค่า | Simplify UX + add incentive |
| เพื่อนบอก "ฟรีก็ใช้ แต่จ่ายไม่จ่าย" | ไม่มี Paid value | เพิ่ม Paid features ที่มีคุณค่าจริง |
| ลูกทริปบ่น "ดู PDF ง่ายกว่า" | UX ไม่ดีพอ | Redesign guest view |

#### 3.2.3 Posts / Info Website Strategy (ช่องทาง Acquisition ใหม่)

**แนวคิด:** Posts เป็นระบบโพสต์ทัวร์แพ็คเกจ แสดงบน Info Website (พัฒนาโดย Nes) เพื่อดึง traffic จาก Google

**Posts กับ Trips เป็นอิสระจากกัน:**
- **Posts** = ดึงคนเข้ามาในแพลตฟอร์ม (Acquisition)
- **Trips** = ให้บริการลูกค้าที่เข้ามาแล้ว (Service)

| กลุ่มเป้าหมาย | ทำไมต้อง Posts บน NatGan? |
|---|---|
| **บริษัททัวร์** | แข่งโปรโมตทัวร์บนแพลตฟอร์ม NatGan แทนสร้างเว็บเอง |
| **ไกด์อิสระ** | ไม่มีงบทำเว็บ → ใช้ Posts เป็นหน้าร้านออนไลน์ showcase ทัวร์ |
| **ผู้ค้นหาทัวร์ (B2C)** | ค้นหาทัวร์ใน Google → เจอ Info Website → ดู Posts → ติดต่อบริษัท |

**Flow:**
```
User ค้นหา "ทัวร์ญี่ปุ่น ซากุระ 2569" ใน Google
     │
     ▼
เจอ Info Website (NatGan) ใน search results
     │
     ▼
ดู Posts ทัวร์แพ็คเกจจากหลายบริษัท
     │
     ▼
เลือกแพ็คเกจ → ติดต่อบริษัท
     │
     ▼
บริษัทใช้ Trip Builder สร้างทริป → ส่ง link ให้ลูกค้า
```

## 9. Risk & Contingency

| Risk | โอกาส | แผนรับมือ |
|---|---|---|
| **Pilot company ไม่ใช้จริง** | กลาง | หา pilot company 2 → มี backup |
| **Free users ไม่ซื้อทริปเพิ่ม** | สูง | ปรับ Paid value proposition + ลดราคาช่วง early + พิจารณา subscription |
| **Facebook Groups ไม่ให้โพสต์** | กลาง | เน้น value content > promo + สร้างกลุ่มเอง |
| **Ads ไม่ work (CAC สูง)** | กลาง | Shift budget ไป content + referral + Posts |
| **ไม่มีใครคลิก Powered by** | ต่ำ-กลาง | A/B test badge design + placement |
| **คนไทยไม่อยากจ่ายค่า per-trip** | สูง | เน้น ROI: "฿49 vs เสียลูกค้า 1 คน" + เพิ่ม free tier value + พิจารณา subscription |
| **Posts ไม่มีคนมาโพสต์** | กลาง | Seed content เอง + ช่วยไกด์สร้าง Posts แรก + incentive ทริปฟรี |
| **Info Website ไม่ติด Google** | กลาง | ลงทุน SEO + structured data + content quality |

## 10. สรุป

### GTM ของ NatGan = 4 เครื่องยนต์

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│   Engine 1: Viral Loop (ฟรี)                                    │
│   ├── ทุก trip link มี "Powered by NatGan"                      │
│   ├── ลูกทริป 30 คน/ทริป x 10 ทริป/เดือน = 300 คนเห็น          │
│   └── 2-3% convert = 6-9 signups/เดือน ฟรี                     │
│                                                                 │
│   Engine 2: Posts / Info Website (ฟรี)                          │
│   ├── บริษัท/ไกด์โพสต์ทัวร์แพ็คเกจ → Google ดึง traffic         │
│   ├── ผู้ค้นหาทัวร์เจอ → ติดต่อบริษัท → บริษัทใช้ Trip Builder   │
│   └── ยิ่งมี Posts มาก → SEO ยิ่งดี → traffic ยิ่งเยอะ           │
│                                                                 │
│   Engine 3: Referral (ต้นทุนต่ำ)                                │
│   ├── ลูกค้าแนะนำเพื่อนในวงการ                                  │
│   ├── ทั้งคู่ได้ทริปฟรี 1 ทริป                                   │
│   └── CAC = ~49 บ/account (ค่าทริปที่ให้ฟรี)                    │
│                                                                 │
│   Engine 4: Content + Ads (ลงทุน)                               │
│   ├── Blog + SEO สำหรับ long-term organic                       │
│   ├── Facebook Ads สำหรับ short-term boost                      │
│   └── CAC = ~600 บ/account                                     │
│                                                                 │
│   รวม 4 engines = 45-80 signups/เดือน @ Phase 3                │
│   Target: 100+ accounts ใน 12 เดือน                             │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**กุญแจสำเร็จ:**
1. **Pilot ต้องสำเร็จ** — ถ้าเพื่อน 1 คนใช้แล้วชอบ ทุกอย่างจะง่ายขึ้น
2. **Powered by badge ต้องมีคนคลิก** — นี่คือ growth engine ที่ CAC = 0
3. **Posts ต้องมี content ดี** — Info Website ต้องติด Google เพื่อดึง traffic
4. **฿49/ทริป ต้อง feel cheap** — ลูกค้าต้องรู้สึกว่า "แค่นี้เอง"
5. **Content ต้อง authentic** — ใช้เรื่องจริง คนจริง ไม่ใช่ marketing speak

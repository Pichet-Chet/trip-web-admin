Data Flow: Create Trip
1 User fills form -> [ถัดไป]
POST /api/upload/image (cover image)
-> returns { url: "https://cdn.natgan.com/..." }
2 Create trip with auto-generated days
POST /api/trips body: { title, destination, dates, ... }
-> Server auto-creates N days from date range
-> returns trip object with empty days[]
3 Redirect to editor
-> /dashboard/trips/[id]/edit
Data Flow: Edit & Notify
1 User edits activity (e.g., time 08:30 -> 09:00)
PUT /api/trips/:dayId/activities/:actId
-> Server: update + increment edit_count + create noti_change_logs
2 Automatic notification dispatch
POST /api/notify/send/:changelogId
-> Query followers -> Send LINE push + Web Push
-> Returns: { sent: { line: 18, web_push: 12 } }
3 Follower receives & acknowledges
LINE: Tap link -> /t/slug -> [รับทราบ] -> POST /api/acknowledge/:c
Web Push: Click -> /t/slug -> [รับทราบ] -> POST /api/acknowledge/:c
Data Flow: Follow via LINE
1 Guest View -> [ติดตามทริปนี้] -> [LINE]
Redirect to LINE Add Friend: line.me/R/ti/p/@natgan-oa
2 LINE Platform sends "follow" event
POST /api/follow/line/webhook
-> Verify signature -> Create noti_followers entry
-> Reply: "สวัสดี! กำลังติดตามทริป: Tokyo Winter 2026"
3 Follower is now subscribed

1. Register & Setup

2. Create Trip

3. Preview & Publish

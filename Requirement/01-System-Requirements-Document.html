System Requirements Document

NatGan (นัดกัน) — Trip Communication Platform
Version 2.0 29 March 2026 Author: Coex Team Status: Draft
1 Executive Summary
แพลตฟอร์มนี้ไม่ใช่ Trip Planner แต่คือ Trip Communication Tool

NatGan คือแพลตฟอร์ม SaaS สำหรับบริษัททัวร์ ไกด์อิสระ และผู้ใช้ส่วนตัว ที่ช่วยสร้าง จัดการ และสื่อสาร itinerary กับลูกทริปแบบ real-time โดยเน้นแก้ปัญหาหลักคือ "แก้ plan แล้วลูกทริปไม่รู้"

นอกจากนี้ยังรองรับระบบ Posts สำหรับโปรโมตแพ็คเกจทัวร์ และระบบ Member สำหรับลูกทริปที่ต้องการเก็บประวัติ

2 Problem Statement

2.1 สถานการณ์ปัจจุบัน

บริษัททัวร์และไกด์อิสระในประเทศไทยส่วนใหญ่ใช้ LINE Group Chat, PDF / Canva, และ โทรศัพท์ ในการสื่อสาร itinerary

2.2 Pain Points หลัก

ปัญหาผลกระทบความถี่
ลูกทริปไม่เห็นข้อความเปลี่ยนแปลง planไปผิดที่ ไปผิดเวลาทุกทริปที่มีการเปลี่ยน
ข้อมูลจมในแชท Lineลูกทริปหาข้อมูลไม่เจอ ต้องถามซ้ำทุกวันระหว่างเดินทาง
ไม่รู้ว่าใครเห็นข้อมูลแล้วไกด์ต้องโทรตามทุกคนทุกครั้งที่มีการเปลี่ยน
ส่ง itinerary ทีละกลุ่ม ทีละคนเสียเวลา ข้อมูลไม่ consistentทุกทริป
ไม่มีช่องทางโปรโมตแพ็คเกจทัวร์เข้าถึงลูกค้าใหม่ยากตลอด

2.3 ข้อจำกัดของ Solution ที่มีในตลาด

Solutionข้อจำกัด
AI Gen Tripสวยแต่แก้ไขไม่ได้, ไม่มี notification, ไม่รองรับ B2B
LINE Groupข้อมูลจมในแชท, ไม่มี read receipt สำหรับ itinerary
PDF / CanvaStatic, แก้ทีต้องส่งใหม่, ไม่มี notification
LINE OAค่าใช้จ่ายสูง (1,500+ บาท/เดือน), ไม่ได้ออกแบบมาเพื่อ itinerary
3 Solution Overview

3.1 Core Concept

บริษัททัวร์/ไกด์ สร้าง Trip Plan บนแพลตฟอร์ม
  → ได้ URL + QR Code
    → ส่งให้ลูกทริปผ่าน Line (เป็น link)
      → ลูกทริปเปิดดูได้เลย ไม่ต้องสมัคร
        → เมื่อ plan เปลี่ยน → LINE Messaging API แจ้งอัตโนมัติ
          → ลูกทริปกด "รับทราบ"
            → บริษัทเห็นว่าใครรู้แล้ว ใครยังไม่รู้

3.3 User Roles

Roleคำอธิบายการเข้าถึง
Admin Ownerสิทธิ์เต็ม: สร้าง แก้ไข ลบ จัดการ trip plan, billing, สมาชิกทีมLogin ผ่าน email/social
Admin Editorสร้าง แก้ไข trip plan ได้ แต่ไม่สามารถลบทริปหรือจัดการ billingLogin, ได้รับเชิญจาก Owner
Guestดู trip plan ผ่าน URL/QRไม่ต้อง login
Memberดู trip plan + มี dashboard ส่วนตัวสมัครผ่าน Google / LINE / Email
Followerดู + รับ notification + กดรับทราบเพิ่มเพื่อน LINE OA / Web Push
Super Adminจัดการระบบ users billingAdmin panel (trip-web-staff)

3.4 Account Types

ประเภทบัญชีคำอธิบายข้อกำหนด
Company (บริษัททัวร์)บริษัททัวร์ที่จดทะเบียนอย่างเป็นทางการต้องมีใบอนุญาต TAT
Freelance Guideไกด์นำเที่ยวอิสระไม่ต้องมี TAT license
Personalผู้ใช้ทั่วไปที่วางแผนทริปกับเพื่อนไม่ต้องมีข้อกำหนดพิเศษ
4 Functional Requirements

4.1 Module: Authentication & Account Management

FR-AUTH-001: การสมัครสมาชิก

  • สมัครผ่าน Email + Password
  • สมัครผ่าน Social Login (Google, LINE)
  • ยืนยัน email ก่อนใช้งาน
  • เลือกประเภทบัญชี: Company / Freelance Guide / Personal
  • รองรับ Free tier อัตโนมัติเมื่อสมัคร

FR-AUTH-002: Company Profile

  • ตั้งชื่อบริษัท/ชื่อไกด์ + อัปโหลด Logo
  • ใส่ช่องทางติดต่อ: โทรศัพท์, LINE ID, Facebook, Instagram, Website
  • ใส่ข้อมูลใบอนุญาต TAT

FR-AUTH-003: Subscription Management

  • ดู plan ปัจจุบัน, ซื้อ Pack เพิ่ม, ดูประวัติการชำระเงิน

4.2 Module: Trip Plan Builder (Admin)

FR-PLAN-001: สร้าง Trip Plan

  • ตั้งชื่อทริป, กำหนดวันเริ่ม-สิ้นสุด, destination, cover image
  • จำนวนผู้เดินทาง, ข้อมูลสายการบิน, ที่พัก
  • เลือกภาษาหลักของ plan (ไทย / อังกฤษ / ญี่ปุ่น)

FR-PLAN-002: จัดการรายวัน (Day)

  • เพิ่ม/ลบ/เรียงลำดับวัน, ชื่อวัน + subtitle, Cover image, Activities

FR-PLAN-003: จัดการกิจกรรม (Activity)

  • เวลา, ชื่อสถานที่, รายละเอียด, Google Maps link, ประเภท, รูปภาพ

FR-PLAN-004: ข้อมูลเสริม

  • เบอร์ฉุกเฉิน, วลีที่ใช้บ่อย, หมายเหตุสำคัญ, ของที่ต้องเตรียม

FR-PLAN-005: Route Planning

  • สร้าง route จาก activities → Google Maps route link อัตโนมัติ

FR-PLAN-006: Template System

  • บันทึก trip plan เป็น template, สร้างใหม่จาก template

4.3 Module: Publishing & Sharing

FR-PUB-001: Publish Trip Plan

  • ปุ่ม Publish → unique URL: natgan.com/t/{slug} + QR Code

FR-PUB-002: Guest View

  • เปิด URL → เห็น trip plan ทันที ไม่ต้อง login
  • Logo + ช่องทางติดต่อ, "Powered by NatGan", ปุ่ม "ติดตามทริปนี้"
  • Responsive design, Auto-highlight กิจกรรมปัจจุบันตามเวลา

FR-PUB-003: Immigration-Friendly View (ยื่น ตม.)

  • แสดงข้อมูลแบบ official: ชื่อทริป, วันเดินทาง, ที่พัก, เที่ยวบิน, Itinerary รายวัน
  • แสดงเป็นภาษาท้องถิ่น destination อัตโนมัติ
  • ไม่ต้องปริ้นกระดาษ — ยื่นหน้าจอให้ ตม. จบ
  • ใช้ offline ได้ (cache ข้อมูล)

FR-PUB-004: Unpublish / Archive

  • Unpublish → URL แสดง "ทริปนี้ไม่เปิดให้ดูแล้ว", Archive → URL ยังเข้าได้ (portfolio)

4.4 Module: Notification & Communication (Killer Feature)

FR-NOTI-001: Follow / Subscribe ทริป

  • LINE OA — User เพิ่มเพื่อน LINE OA ของ NatGan → link trip
  • Web Push — กด allow บน browser ไม่ต้องมี account

FR-NOTI-002: แจ้งเตือนเมื่อ Plan เปลี่ยน (Core)

⚠️ ทริป "เชียงใหม่ 3 วัน" มีการเปลี่ยนแปลง

- เพิ่ม: ร้านกาแฟ Ristr8to 10:00
- ยกเลิก: ตลาดวโรรส (ปิดปรับปรุง)

👉 ดูรายละเอียด: natgan.com/t/chiangmai3d

FR-NOTI-003: แจ้งเตือนก่อนวันถัดไป (Auto)

📋 พรุ่งนี้ วันที่ 3: ภูเขาไฟฟูจิ

⏰ นัดพบ 07:00 ล็อบบี้โรงแรม
🎒 เตรียม: เสื้อกันหนาว, รองเท้าผ้าใบ

👉 ดูรายละเอียด: natgan.com/t/tokyo8d

FR-NOTI-004: ปุ่มรับทราบ (Acknowledge)

  • Banner บน guest view + ปุ่ม "รับทราบแล้ว" → บันทึกว่าเห็นแล้ว

FR-NOTI-005: Read Receipt Dashboard (Admin)

การเปลี่ยนแปลงวันที่ 15 มี.ค. 2026
✅ รับทราบ: 28/30 คน
❌ ยังไม่เห็น: สมชาย, สมหญิง
  • ปุ่ม "ส่ง noti ซ้ำ" เฉพาะคนที่ยังไม่เห็น

4.5-4.6 Module: Change Log & Dashboard

FR-LOG-001: ประวัติการเปลี่ยนแปลง

  • บันทึก change log ทุกครั้งที่ plan ถูกแก้ไข — แสดงบน admin dashboard

FR-DASH-001: Admin Dashboard

  • รายการ trip plans (Active/Draft/Archived), views, followers, การเปลี่ยนแปลงล่าสุด

FR-DASH-002: Trip Analytics (Pro/Business)

  • Views ต่อวัน, followers, acknowledge rate, QR scans

4.7-4.8 Module: Branding & Billing

FR-BRAND: Company Branding + Powered By

  • Free tier: "Powered by NatGan" ถาวร
  • Pro: badge เล็ก / Business: ซ่อนได้ (white-label)

FR-BILL-001: Pricing Tiers

Tierราคารายละเอียด
Freeฟรี3 ทริป
Pay-per-Trip฿49/ทริปซื้อทีละทริป
Pack 5฿1995 ทริป (ประหยัด ฿46)
Pack 10฿34910 ทริป (ประหยัด ฿141)

FR-BILL-002: Payment Methods

  • PromptPay QR (หลัก), Credit/Debit card ผ่าน Omise, ออกใบเสร็จอัตโนมัติ

4.9 Module: Posts (Tour Package Listings)

ระบบ Posts และ Trips เป็นระบบอิสระต่อกัน ไม่ได้เชื่อมโยงกัน

FR-POST-001: สร้าง Post

  • title, destination, description, highlights, images, priceStartFrom, duration, travelPeriod, totalSlots, slotsLeft, tags

FR-POST-002: สถานะ Post

  • Draft → Published (แสดงบน Info Website) → Closed

FR-POST-003: การแสดงผล

  • ไม่มีระบบจอง (booking) — showcase เท่านั้น บริษัท/ไกด์ปิดการขายเอง

4.10 Module: Client Member System

  • สมัครผ่าน Google, LINE, Email (optional)
  • Member Dashboard: ทริปที่ติดตาม, สถานที่ที่บันทึก, ประวัติทริป
  • Travel Points / Elite Member — planned (TBD)

4.11-4.13 Module: Portfolio, Support Ticket, Multi-language

FR-PORT: Portfolio สาธารณะ

  • URL: natgan.com/p/{custom-slug} — แสดงทริปที่ published + ข้อมูลบริษัท

FR-TICKET: Support Ticket

  • แจ้งบัก / ร้องขอฟีเจอร์ — สถานะ: Open → Pending → Resolved

FR-I18N: Multi-language

  • Platform UI: ไทย, อังกฤษ, ญี่ปุ่น
  • Trip Plan: เลือกภาษาหลัก (อนาคต: หลายภาษาใน 1 plan)
